# 엑셀 파일 구조 (학습리포트_정리본.xlsx)

> 이 문서가 있으면 AI에게 .xlsx 파일을 직접 줄 필요 없음.
> 엑셀 파싱/저장 작업 시 이 파일만 참조하면 됨.

---

## 시트 목록

| 시트명 | 역할 |
|---|---|
| `수업정보` | 날짜별 수업 내용 및 과제 |
| `성적` | 학생별 날짜별 점수/오답 |
| _(학생별 시트)_ | 학생 이름과 동일, 과제 이행 현황 |

---

## 시트 1: 수업정보

**헤더 행 (0행):**

| 열 인덱스 | 열 이름 | 타입 | 예시 |
|---|---|---|---|
| 0 | 날짜 | Date / 숫자코드 | `2026-03-03` 또는 엑셀 날짜코드 |
| 1 | 강사명 | string | `김선생` |
| 2 | 교재 | string | `수학의 정석` |
| 3 | 단원 | string | `2단원 함수` |
| 4 | 상세진도 | string | `p.45~52 예제풀이` |
| 5 | 과제1 | string | `p.53 연습문제 1~5` |
| 6 | 과제2 | string | |
| 7 | 과제3 | string | |
| 8 | 과제4 | string | |
| 9 | 과제5 | string | |

**특수 행:**
- `[전체문제수]` 텍스트가 포함된 행: 날짜열에 특정 날짜, 강사명열에 숫자(전체문제수)
  - `parseWB`에서 `G.lessons[i].전체문제수`에 저장

**정렬:** 날짜 오름차순 (`toDS()` 통해 정규화 후 sort)

---

## 시트 2: 성적

**구조:** 학생 1명당 1행, 날짜별 컬럼이 동적으로 추가됨

| 열 패턴 | 내용 |
|---|---|
| `이름` (고정, 0열) | 학생 이름 |
| `YYYY-MM-DD_맞힌개수` | 해당 날짜 맞힌 문제 수 (숫자) |
| `▼오답` (구분자) | 오답 섹션 시작 표시 |
| `YYYY-MM-DD_오답` | 해당 날짜 오답 번호 문자열 (예: `"2,4,7"`) |

**파싱 결과:**
- `G.students[]` — 이름 목록
- `G.corrects[학생명][날짜]` — 맞힌 개수
- `G.wrong[학생명][날짜]` — 오답 번호 문자열

---

## 학생별 시트 (예: `홍길동`)

**1행:** 헤더

| 열 인덱스 | 내용 |
|---|---|
| 0 | 날짜 |
| 1 | 이행률 (%) |
| 2 | 과제1 내용 |
| 3 | 과제1 상태 (`완료` / `부분완료` / `미완료` / 빈값) |
| 4 | 과제2 내용 |
| 5 | 과제2 상태 |
| 6 | 과제3 내용 |
| 7 | 과제3 상태 |
| 8 | 과제4 내용 |
| 9 | 과제4 상태 |
| 10 | 과제5 내용 |
| 11 | 과제5 상태 |

**특수 행:** 날짜열에 `[과제]` 포함된 행 → 해당 날짜의 과제 내용 헤더 행

**파싱 결과:**
- `G.hwRec["학생명||날짜"]` = `{이행률, 과제1_상태, ..., 과제5_상태}`
- `G.rates[학생명][날짜]` = 이행률 숫자

---

## 날짜 형식 정규화 규칙 (`toDS` 함수)

```
입력: "2026/03/03"  → "2026-03-03"
입력: "2026-03-03"  → "2026-03-03"
입력: 46083 (엑셀코드) → XLSX.SSF.parse_date_code() → "2026-03-03"
규칙: replace('/', '-') 후 substring(0,10)
```
