# 엑셀 파일 구조

> 이 문서가 있으면 AI에게 .xlsx 파일을 직접 줄 필요 없음.
> 엑셀 파싱/저장 작업 시 이 파일만 참조.

**파일명:** 사용자가 로드한 파일명 그대로 사용 (`G.excelFileName`에 저장됨).
기본값 `학습리포트_데이터.xlsx`이나, 로드 즉시 실제 파일명으로 교체됨.

---

## 시트 목록

| 시트명 | 역할 |
|---|---|
| `수업정보` | 날짜별 수업 내용 및 과제 |
| `성적` | 학생별 날짜별 맞힌개수/오답 |
| _(학생 이름과 동일한 시트)_ | 학생별 과제 이행 현황 |

---

## 시트 1: 수업정보

**헤더 행 (0행):**

| 열 인덱스 | 열 이름 | 타입 | 예시 |
|---|---|---|---|
| 0 | 날짜 | Date / 엑셀코드 | `2026-03-03` 또는 숫자코드 |
| 1 | 강사명 | string | `김선생` |
| 2 | 교재 | string | `수학의 정석` |
| 3 | 단원 | string | `2단원 함수` |
| 4 | 상세진도 | string | `p.45~52 예제풀이` |
| 5~9 | 과제1~과제5 | string | `p.53 연습문제 1~5` |

**특수 행:**
- `[전체문제수]` 텍스트가 포함된 행: 강사명 열에 숫자(전체문제수) 입력
  - `parseWB`에서 `G.lessons[i].전체문제수`에 저장
  - 해당 날짜의 미니테스트 전체 문항 수로 사용

**정렬:** 날짜 오름차순 (`toDS()` 정규화 후 sort)

---

## 시트 2: 성적

**구조:** 학생 1명당 1행, 날짜별 컬럼이 동적으로 추가됨

| 열 패턴 | 내용 |
|---|---|
| `이름` (고정, 0열) | 학생 이름 |
| `YYYY-MM-DD_맞힌` | 해당 날짜 맞힌 문제 수 (숫자) |
| `▼오답` (구분자) | 오답 섹션 시작 표시 |
| `YYYY-MM-DD_오답` | 해당 날짜 오답 번호 문자열 (예: `"2,4,7"`) |

**특수 행:**
- `[전체문제수]` 행: 날짜별 전체 문항 수 → `G.lessons[].전체문제수` 갱신

**파싱 결과:**
- `G.students[]` — 이름 목록 (등장 순서)
- `G.corrects[학생명][날짜]` — 맞힌 개수
- `G.wrong[학생명][날짜]` — 오답 번호 문자열

---

## 학생별 시트 (예: `홍길동`)

**헤더 행 (0행):**

| 열 인덱스 | 내용 |
|---|---|
| 0 | 날짜 |
| 1 | 이행률 (%) |
| 2 | 과제1 내용 |
| 3 | 과제1 상태 (`완료` / `부분완료` / `미완료` / 빈값) |
| 4~11 | 과제2~5 내용/상태 (동일 패턴) |

**파싱 결과:**
- `G.hwRec["학생명||날짜"]` = `{이행률, 과제1_상태, ..., 과제5_상태}`
- `G.rates[학생명][날짜]` = 이행률 숫자

---

## 날짜 형식 정규화 (`toDS`)

```
입력: "2026/03/03"  → "2026-03-03"  (슬래시 → 하이픈, 10자 슬라이스)
입력: "2026-03-03"  → "2026-03-03"  (그대로)
입력: 46083 (엑셀코드) → XLSX.SSF.parse_date_code() → "2026-03-03"
```

## saveToExcel 저장 구조

`G → 엑셀` 방향으로 동일한 3종 시트 구조로 재생성.
저장 파일명: `G.excelFileName` (로드 시 파일명 그대로).
