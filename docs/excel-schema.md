# 엑셀 파일 구조 (학습리포트_데이터.xlsx)

> 이 문서가 있으면 AI에게 .xlsx 파일을 직접 줄 필요 없음.
> 엑셀 파싱/저장 작업 시 이 파일만 참조하면 됨.

---

## 시트 목록 (신규 형식)

| 시트명 | 역할 |
|---|---|
| `수업정보` | 날짜별 수업 내용 및 과제 |
| `YYYY-MM-DD` _(날짜별)_ | 해당 날짜 학생 전체의 성적·과제이행 현황 |

> **하위 호환:** 구 형식(성적 시트 + 학생별 시트)도 `parseWB`에서 읽기 지원.

---

## 시트 1: 수업정보

**헤더 행 (0행):**

| 열 인덱스 | 열 이름 | 타입 | 예시 |
|---|---|---|---|
| 0 | 날짜 | Date / 숫자코드 | `2026-03-03` 또는 엑셀 날짜코드 |
| 1 | 강사명 | string | `김선생` |
| 2 | 교재 | string | `수학의 정석` |
| 3 | 단원 | string | `2단원 함수` |
| 4 | 상세진도 | string | `p.45~52 예제풀이` |
| 5 | 과제1 | string | `p.53 연습문제 1~5` |
| 6 | 과제2 | string | |
| 7 | 과제3 | string | |
| 8 | 과제4 | string | |
| 9 | 과제5 | string | |

**정렬:** 날짜 오름차순 (`toDS()` 통해 정규화 후 sort)

## 시트 2~N: 날짜별 시트 (예: `2026-03-09`)

**행 구조:**

| 행 | 역할 |
|---|---|
| 0 | 헤더 (`이름`, `성적`, `오답`, `과제이행률`, `과제1`~`과제5`) |
| 1 | 과제 내용행 (열 4~8에 해당 날짜의 과제1~5 내용 표시, 수업정보에서 자동 채움) |
| 2+ | 학생별 데이터 |

**열 구조 (학생 데이터 행):**

| 열 인덱스 | 열 이름 | 내용 |
|---|---|---|
| 0 | 이름 | 학생 이름 |
| 1 | 성적 | `맞힌수/전체문제수` 형식 (예: `5/5`, `3/5`) |
| 2 | 오답 | 오답 번호 문자열 (예: `"2,4"`) |
| 3 | 과제이행률 | 숫자 (0~100) |
| 4 | 과제1 | 과제1 상태 (`완료` / `부분완료` / `미완료` / 빈값) |
| 5 | 과제2 | 과제2 상태 |
| 6 | 과제3 | 과제3 상태 |
| 7 | 과제4 | 과제4 상태 |
| 8 | 과제5 | 과제5 상태 |

**예시 시트 (2026-03-09):**

```
이름    성적   오답   과제이행률   과제1     과제2     과제3   과제4   과제5
                      p.9-14    p.17-19   곱셈공식써오기
하정    5/5          100        ○         △         X
김도은  5/5          80         ○         ○
윤희    3/5          60         ○
황선재  2/5          40
원빈    1/5    2,4   20
희은    5/5          100        ○         ○         ○       ○
```

**파싱 결과:**
- `G.students[]` — 첫 번째 날짜 시트 기준으로 학생 순서 결정
- `G.corrects[학생명][날짜]` — 맞힌 개수
- `G.wrong[학생명][날짜]` — 오답 번호 문자열
- `G.rates[학생명][날짜]` — 과제이행률
- `G.hwRec["학생명||날짜"]` = `{이행률, 과제1_상태, ..., 과제5_상태}`

## 날짜 형식 정규화 규칙 (`toDS` 함수)

```
입력: "2026/03/03"  → "2026-03-03"
입력: "2026-03-03"  → "2026-03-03"
입력: 46083 (엑셀코드) → XLSX.SSF.parse_date_code() → "2026-03-03"
규칙: replace('/', '-') 후 substring(0,10)
```

---

## saveToExcel 저장 구조

`G → 엑셀` 방향으로 두 가지 시트 타입으로 재생성.

1. `수업정보` 시트: G.lessons 전체
2. 날짜별 시트: G.lessons의 각 날짜마다 1개, G.students 전체 포함

기존 파일명 `G.excelFileName` 기준으로 다운로드.

---

## 학생 관리

- `G.students[]` 가 마스터 목록
- UI의 **학생 목록** 섹션에서 추가/삭제 가능
- 저장 시 현재 `G.students` 기준으로 모든 날짜 시트 자동 재생성
- `createTemplate()` 호출 시 오늘~6월까지 주 1회 날짜 + 기본 학생 6명으로 템플릿 생성

## 구 형식 (하위 호환)

시트명이 날짜 형식(`YYYY-MM-DD`)이 아닌 파일은 구 형식으로 간주:
- `성적` 시트: 날짜별 맞힌수·오답 데이터
- 학생 이름 시트: 과제이행 기록
